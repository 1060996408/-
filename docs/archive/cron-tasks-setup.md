## 🔄 工作流程

### 每日维护流程 (23:50)
```
1. auto-updater
   └─ 检查 OpenClaw 核心更新
   └─ 检查所有 skills 更新
   └─ 下载并安装更新

2. skill-vetter
   └─ 扫描新安装的代码
   └─ 检查是否有恶意行为
   └─ 生成安全审计报告

3. self-improving
   └─ 分析今日所有对话
   └─ 提取经验教训
   └─ 生成反思报告

4. GitHub 提交
   └─ 整理更新日志
   └─ 提交到仓库 1060996408/-
   └─ Push 到 master 分支
```

### PR 审查流程 (每 30 分钟)
```
1. 检查所有 fix/issue-* 分支的 PR
2. 抓取新的 review 评论
3. 分析评论内容 (是否需要修改)
4. 如果需要修改:
   └─ 自动修改代码
   └─ 运行测试
   └─ 提交并推送
   └─ 回复评论
5. 如果没有新评论:
   └─ 静默退出
```

### 健康检查流程 (08:00)
```
1. 运行 openclaw doctor
2. 检查项目:
   └─ Node.js 版本
   └─ Skills 完整性
   └─ 配置文件有效性
   └─ 网关端口安全性
   └─ 潜在的后门代码
3. 生成诊断报告
4. 如果发现问题,发送警告
```
